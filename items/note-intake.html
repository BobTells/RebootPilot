Note Intake

Build an email-based note intake setup that lands in a vault or project without turning setup into a second job. This version is the live launch piece inside RebootPilot.

Medium

Install an app, connect an account, and tune the workflow once reality starts poking holes in it.

Core Tools
  • Claude Desktop, Antigravity, or Codex
  • Gmail or Outlook
  • Google Drive or OneDrive
Optional
  • Power Automate for Outlook intake
  • Obsidian or another markdown viewer
  • Trigger-based upstream automations
What it does

Email-driven intake into a vault or AI project. The folder builder is just a starting shape, not the final law of the land.

Build your setup

Choose the email path, routing behavior, structure approach, and note logic. The tool will generate copy-ready prompts for your selected runtime.

1. Email path
2. Routing mode
3. Filing logic
4. Runtime
5. Destination
6. File setup

Start with top-level folders, then nest deeper only where needed. Each branch can have up to 2 children and go up to 8 levels deep.

Recommended path: start with Inbox + AI assist and a flatter structure. If the vault starts drifting, use the assistant to refine naming, branching, and tag rules instead of trying to over-design everything up front.
